03.02 — Il CLR (Virtual Machine)

Il CLR (Common Language Runtime)

È la Virtual Machine di .NET. Gestisce l'intero ciclo di vita dell'applicazione: eccezioni, sicurezza, thread e gestione della memoria.

Compilatore JIT (Just-In-Time)

Traduce l'IL (Intermediate Language) in codice macchina reale solamente quando serve, ottimizzando le performance per l'hardware specifico.

03.03 — Gestione Memoria

Garbage Collector

  • Individua oggetti non più raggiungibili.
  • Libera la memoria automaticamente (Heap gestito).

Addio manuale

A differenza di C++, non dobbiamo usare free() o delete. Il GC previene memory leaks e dangling pointers.

03.04 — Heap Generazionale

L'Heap è diviso in tre generazioni basate sull'idea che gli oggetti nuovi muoiano presto.

Gen 0

Oggetti appena creati (Short-lived).

Gen 1

Buffer tra oggetti nuovi e longevi.

Gen 2

Oggetti a lungo termine (Long-lived).

Compilazione

Compilati, Interpretati e l'Approccio .NET

06.01 — Linguaggi Compilati (es. C, C++)

Traduzione Totale

Tutto il codice viene tradotto in binario prima dell'esecuzione. Il risultato è un file eseguibile pronto per il processore.

Massima Efficienza a Runtime

Blocco Totale

Dimentichi un ";"?
La compilazione fallisce.
Niente eseguibile finché tutto non è perfetto.

06.02 — Linguaggi Interpretati (es. JS, Python)

Traduzione in Tempo Reale

Il codice viene letto, tradotto ed eseguito riga per riga durante l'esecuzione. Non esiste un file eseguibile preventivo.

Flessibile e Immediato da Eseguire

Fragilità a Runtime

L'esecuzione continua finché non trova l'errore.
Ciò che sta "sotto" non viene mai eseguito.

12.01 — Il dogma dell'Immutabilità

Sola Lettura

Le stringhe in C# sono immutabili. Una volta create nello Heap, non possono più cambiare.

Ogni "modifica" crea una nuova stringa.
La stringa originale resta intatta nello Heap.
Memory Flow
Program.cs
1
2
string s = "Ciao"; // → crea String Object A nello Heap
s += " Mondo"; // → crea Object B — A viene abbandonato
Attenzione: manipolazioni massive in loop possono saturare la memoria — usare StringBuilder.
